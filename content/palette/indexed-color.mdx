## インデックスカラーモードとは

通常の<Term id="rgba">RGBA</Term>モードでは、各ピクセルが「赤255・緑128・青0」のように**色そのもの**を持っています。

一方、<Term id="index-color">インデックスカラー</Term>モードでは、各ピクセルが持つのは**パレットの番号（インデックス）**だけです。「このピクセルはパレットの3番の色」という情報しかありません。

この仕組みにより、**パレットの色を変更すると、その番号を参照しているすべてのピクセルの見た目が一括で変わります**。

## RGBAモードとの違い

|                                             | RGBA                 | インデックスカラー             |
| ------------------------------------------- | -------------------- | ------------------------------ |
| ピクセルが持つ情報                          | 色の値（RGBA）       | パレット番号（0〜255）         |
| パレット色を変更                            | キャンバスに影響なし | **全ピクセルに即反映**         |
| 使える色数                                  | 約1677万色           | 最大256色                      |
| 半透明                                      | 使える               | 使えない（透明 or 不透明のみ） |
| <Term id="blend-mode">ブレンドモード</Term> | 使える               | 使えない                       |

## 使い方

### 新規作成時に選ぶ

1. **File > New**（`Ctrl+N`）で新規作成
2. **Color Mode** を **Indexed** に設定
3. OK で作成

### 既存スプライトを変換する

すでにRGBAモードで描いた<Term id="sprite">スプライト</Term>をインデックスカラーに変換することもできます。

1. **Sprite > Color Mode > Indexed** を選択
2. 自動的に現在のパレットに合わせて色が変換される

> 256色を超える色を使っている場合、近似色に自動変換されます。事前にパレットを整理しておくときれいに変換されます。

### RGBAに戻す

**Sprite > Color Mode > RGB** で戻せます。ピクセルの色はそのまま保持されるので、見た目は変わりません。

## パレットの色を変えてみよう

インデックスカラーモードの最大の強みを体験してみましょう。

### 手順

1. インデックスカラーモードでキャラクターを描く
2. <Term id="palette">パレット</Term>で肌色をダブルクリック
3. <Term id="color-picker">カラーピッカー</Term>で別の色に変更
4. → キャンバス上の肌色が**すべて一瞬で変わる**

これがインデックスカラーの威力です。

### 活用シーン

- **配色バリエーションの検討** — キャラクターの服の色を何パターンも試せる
- **ゲームの色違いキャラ** — 同じドット絵でパレットだけ差し替えて敵バリエーションを作る
- **雰囲気の調整** — 全体の色味をパレット操作でまとめて変える
- **レトロゲーム風の制約** — ファミコンやゲームボーイ風の色制限を再現

## 透明色の設定

インデックスカラーでは半透明が使えません。代わりに、パレットの特定のインデックスを**透明色**として扱います。

- デフォルトではインデックス **0番** が透明色
- **Sprite > Properties** で透明色のインデックスを変更可能
- 背景<Term id="layer">レイヤー</Term>には透明色は適用されない

> 透明色に設定されたインデックスの色で描画すると、そのピクセルは透明になります。

## パレットスワップテクニック

### 色違いキャラの作り方

ゲーム開発でよく使われるテクニックです。

1. キャラクターをインデックスカラーで描く
2. **パレットを保存**（パレットオプション ☰ → Save Palette）
3. パレットの色を差し替えて「色違いバージョン」を作る
4. **別名で保存**して色違いパレットとして管理

同じスプライトに異なるパレットを適用するだけで、赤チーム・青チームのキャラクターが作れます。

### 昼→夕→夜の演出

背景タイルのパレットを段階的に変えることで、時間経過を表現できます。

1. 通常の背景を描く
2. パレットをコピー
3. 全体を暖色寄りにシフト → 夕暮れパレット
4. 全体を暗く青寄りにシフト → 夜パレット

## Shading Ink との組み合わせ

インデックスカラーモードでは **Shading Ink** が特に強力です。

1. パレットに明→暗の順で色を並べる
2. <Term id="brush">ブラシ</Term>の Ink を **Shading** に変更
3. 左クリックで塗ると**パレット上で1つ明るい方**にシフト
4. 右クリックで**1つ暗い方**にシフト

影やハイライトをパレット順に沿って簡単に塗れるので、インデックスカラー + Shading Ink は定番の組み合わせです。

## 注意点

### 256色の制限

インデックスカラーのパレットは**最大256色**です。複雑なイラストには向きませんが、<Term id="pixel-art">ドット絵</Term>では十分な色数です。

### ブレンドモードが使えない

レイヤーのブレンドモード（Multiply、Screen など）はRGBAモード専用です。インデックスカラーでは Normal のみ使えます。

### 色の自動変換に注意

RGBAからインデックスカラーに変換する際、パレットにない色は最も近い色に変換されます。意図しない色になることがあるので、変換前に確認しましょう。

## 💡 Tips

### RGBAでもパレットロックで似た運用ができる

RGBAモードでも、描画時に **Edit > FX > Remap** を使ったり、パレットから色を選ぶ運用を徹底すれば似たことはできます。ただし「パレット変更で一括変換」はインデックスカラー専用の機能です。

### ゲームボーイ風パレット

4色だけのパレット（白・ライトグリーン・ダークグリーン・黒）を作ってインデックスカラーモードで描くと、本格的なゲームボーイ風ドット絵が作れます。パレットの4色を入れ替えるだけで、ゲームボーイポケット風（グレースケール）にも瞬時に変えられます。

### コマンドラインでパレットスワップ

Aseprite CLI を使えば、パレットの差し替えを自動化できます：

```bash
aseprite -b sprite.aseprite --palette night.gpl --save-as sprite-night.png
```

ゲーム用に複数のカラーバリエーションを一括生成する場合に便利です。
