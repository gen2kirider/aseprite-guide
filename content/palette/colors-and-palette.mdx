## カラーピッカーの使い方

Asepriteには強力な<Term id="color-picker">カラーピッカー</Term>が搭載されています。画面下部のカラーバーをクリックするとカラーピッカーが開きます。

### カラーモデルの切り替え

カラーピッカーでは複数のカラーモデルを使い分けることができます：

- **<Term id="rgba">RGB</Term>** — 赤・緑・青の3色で色を指定。Web色やプログラムとの連携に便利
- **<Term id="hsv">HSV</Term> / HSL** — 色相（Hue）・彩度・明度で指定。直感的に色を選びやすい
- **Hex** — `#FF5733` のような16進数で直接入力

> <Term id="pixel-art">ドット絵</Term>では **HSV**
> モードが特に便利です。同じ色相のまま明度だけ変えることで、自然な影色を作れます。

### スポイトツール（Alt / I）

キャンバス上の色を直接拾うことができます。<Term id="eyedropper">スポイト</Term>はとても頻繁に使うツールです。

- `Alt` + 左クリック → 前景色にセット
- `Alt` + 右クリック → 背景色にセット
- `I` キーでスポイトモードに切り替え

## パレットの基本

パレットは<Term id="pixel-art">ドット絵</Term>の色数を管理するための機能です。画面右側に表示されるカラーグリッドが<Term id="palette">パレット</Term>です。

### パレットの操作

| 操作         | やり方                    |
| ------------ | ------------------------- |
| 色の追加     | パレット下部の `+` ボタン |
| 色の削除     | 色を選択して `-` ボタン   |
| 色の並べ替え | ドラッグ＆ドロップ        |
| 色の編集     | ダブルクリック            |
| 範囲選択     | `Shift` + クリック        |

### インデックスカラーモード

Asepriteでは<Term id="sprite">スプライト</Term>の**カラーモード**を選択できます：

1. <Term id="rgba">RGBA</Term> — フルカラー。自由に色を使える
2. <Term id="index-color">インデックスカラー</Term> —
   パレットの色だけを使用する制限モード
3. グレースケール — 白黒のみ

> インデックスカラーモードでは、パレットの色を変更するとキャンバス上の該当色も**一括で変わります**。カラーパレットの調整がとても楽になります。

## カスタムパレットの作り方

### ランプ（グラデーション）の作成

2色を選択して **Edit > FX > Gradient** を使うと、間を補完する<Term id="ramp">グラデーション</Term>が自動生成されます。

手順：

1. パレット上で開始色をクリック
2. `Shift` を押しながら終了色をクリック（範囲選択）
3. メニューから **Sort & Gradients > Gradient** を選択

### プリセットパレット

Asepriteにはいくつかのパレットが最初から入っています。パレットオプション（☰）→ **Presets** から切り替えられます。

- **Default** — Aseprite標準の256色パレット
- **DB16 / DB32** — DawnBringer氏の16色・32色パレット
- **Pico-8** — 16色の制限パレット。レトロゲーム風に最適
- その他 **CGA**、**Software** など

### 外部パレットの導入

コミュニティで人気の高い以下のようなパレットは、自分でダウンロードして読み込む必要があります：

- **ENDESGA 32 / 36** — Lospec等で配布。ゲーム制作で人気
- **AAP-64** — Adigun A. Polack氏の64色パレット

導入手順：

1. [Lospec](https://lospec.com/palette-list) などから `.gpl` や `.png` 形式でダウンロード
2. パレットオプション（☰）→ **Load Palette** で読み込み
3. **Save Palette** で保存しておけば次回以降も Presets から選べる

### パレットの権利について

「外部パレットを使ってゲームを販売して大丈夫？」という疑問を持つ方もいると思います。結論から言うと、**基本的に問題ありません**。

- パレット（色の組み合わせ）自体は「アイデア」であり、著作権法で保護される「表現」には該当しないのが一般的な解釈です
- さらに、有名パレットの多くは **CC0（パブリックドメイン）** で公開されています

| パレット | ライセンス | 商用利用 | クレジット |
|---------|-----------|---------|-----------|
| ENDESGA 32 / 36 | CC0 | OK | 不要 |
| DB16 / DB32 | CC0 | OK | 不要 |
| AAP-64 | CC0 | OK | 不要 |

Lospec の各パレットページにライセンスが記載されているので、気になる場合は確認しておきましょう。義務ではありませんが、クレジットに「Palette: ENDESGA 32」と一行入れておくと作者へのリスペクトになります。

## 実践：キャラクターの配色

ドット絵キャラクターの色を決めるときのコツです。

### 基本の3段階

1つのパーツにつき **3色** を用意すると立体感が出ます：

1. **ベース色** — パーツの基本色
2. **影色** — ベース色より明度を下げ、色相を少し暖色寄りに
3. **ハイライト** — ベース色より明度を上げ、彩度を少し下げる

### 色数を抑えるコツ

- 1キャラクター **8〜12色** 程度に抑える
- 肌色の影は赤寄り、服の影は青寄りにすると自然
- アウトラインはパーツの影色をさらに暗くした色を使う

```
ハイライト:  HSV(210, 40%, 90%)
ベース色:    HSV(210, 60%, 70%)
影色:        HSV(220, 70%, 45%)
```

## まとめ

- カラーピッカーは **HSV** モードが直感的
- パレットの色数を意識すると統一感のあるドット絵に
- **ランプ機能**でグラデーションを自動生成
- 1パーツ3色（ハイライト・ベース・影）が基本

## 💡 知っておくと差がつくTips

### 既存パレットは「変える」が基本

ENDESGA 32やDB32などの定番パレットを使う場合、色を**増やす**のではなく**差し替え・微調整する**のがプロの使い方です。

- パレットの色数を増やすと統一感が崩れやすい
- 使わない色を必要な色に**入れ替える**
- 色相を5°ずらす、明度を少し変える程度の微調整で十分

> 32色パレットに「あと1色だけ…」を繰り返すと収拾がつかなくなります。色数を一定に保つ制約が、結果的にドット絵全体の色の調和を守ってくれます。

### ゲームジャンル別おすすめパレット

| ジャンル           | おすすめパレット | 理由                                     |
| ------------------ | ---------------- | ---------------------------------------- |
| ゼルダ風アクション | ENDESGA 36       | 緑の階調が豊富でフィールドマップに最適   |
| レトロRPG          | DB32             | 肌色・草・岩・水がバランスよく揃っている |
| レトロアクション   | Pico-8 (16色)    | 制限がレトロ感を演出                     |
| 多彩なマップ       | AAP-64           | 64色でタイルバリエーションに対応         |

### 影色は色相をずらす

影を単に暗くするだけでは平坦な印象になります。影色を作るときは：

- **暖色系のパーツ**（肌・赤い服など）→ 影は**赤〜紫寄り**に
- **寒色系のパーツ**（青い服・金属など）→ 影は**青〜紫寄り**に
- **草・木など**→ 影は**青緑〜暗い緑**に

この「色相シフト」を意識するだけで、ドット絵の立体感が格段に上がります。

### パレットを先に決めてから描く

初心者にありがちなのが「描きながら色を足していく」やり方です。これだと色数が膨らみがちです。

1. まずパレットを決める（既存パレットをロード）
2. パレット内の色だけで描く
3. どうしても足りなければ、使っていない色と**入れ替える**

この順番を守るだけで、完成度がぐっと上がります。
