## スライスとは

ゲームUIのボタンやウィンドウ枠は、サイズが変わっても角が崩れないように **9スライス（9-patch）** で設計します。

Asepriteの**スライス機能**を使うと、<Term id="sprite">スプライト</Term>上に9スライスの境界線を定義し、その情報をJSON付き<Term id="spritesheet">スプライトシート</Term>に書き出せます。

## 9スライスの仕組み

画像を3×3の9つの領域に分割します：

```
┌────┬──────────┬────┐
│ TL │    Top   │ TR │  ← 角はそのまま
├────┼──────────┼────┤
│ L  │  Center  │  R │  ← 中央は引き伸ばし
├────┼──────────┼────┤
│ BL │  Bottom  │ BR │  ← 角はそのまま
└────┴──────────┴────┘
```

- **四隅（TL, TR, BL, BR）** — サイズ変更しても変わらない
- **上下左右（Top, Bottom, L, R）** — 一方向に引き伸ばされる
- **中央（Center）** — 両方向に引き伸ばされる

## スライスの作成

### 手順

1. ツールバーから **スライスツール**（`C`）を選択
2. <Term id="canvas">キャンバス</Term>上でUI素材の範囲をドラッグして選択
3. ダブルクリックで **Slice Properties** を開く
4. **9-Slices** の **Center** の位置を数値で指定

### Center の指定

9スライスの内側の境界を `x, y, width, height` で指定します。

例えば 24×24 のボタンで、角を 4px ずつ残したい場合：

- **Center**: `4, 4, 16, 16`

つまり「左上から4px, 上から4pxの位置に、16×16の中央領域」を定義します。

### プロパティ

| 項目       | 説明                                   |
| ---------- | -------------------------------------- |
| **Name**   | スライスの名前（エクスポート時に使用） |
| **Bounds** | スプライト上の位置とサイズ             |
| **Center** | 9スライスの中央領域                    |
| **Pivot**  | ピボットポイント（任意）               |

## 複数スライスの管理

1つのスプライトに複数のUIパーツがある場合、それぞれにスライスを設定できます。

例：ボタン3種 + ウィンドウ枠 + HP バーをまとめた UI アトラス

1. 各パーツごとにスライスを作成
2. わかりやすい名前を付ける（`btn_normal`, `btn_hover`, `window_frame` など）
3. 一括で<Term id="export">エクスポート</Term>

## エクスポート

### JSON付きスプライトシートで書き出し

**File > Export Sprite Sheet** でスライス情報を含むJSONを出力できます。

書き出されたJSONにスライス情報が含まれます：

```json
{
  "meta": {
    "slices": [
      {
        "name": "btn_normal",
        "color": "#0000ffff",
        "keys": [
          {
            "frame": 0,
            "bounds": { "x": 0, "y": 0, "w": 48, "h": 24 },
            "center": { "x": 4, "y": 4, "w": 40, "h": 16 }
          }
        ]
      }
    ]
  }
}
```

### ゲームエンジンでの利用

**Unity:**

- Sprite Editorの **Custom Outline** や **Border** 設定で9スライスを指定
- AsepriteのJSON からスライス情報を読み取るインポーターもある

**Godot:**

- **NinePatchRect** ノードでスライス画像を設定
- JSONのcenter値をパッチマージンに対応させる

## アニメーションスライス

スライスは<Term id="frame">フレーム</Term>ごとに異なる設定が可能です。ボタンの通常状態・ホバー状態・押下状態をフレームで分けて、各フレームにスライスを設定すれば、アニメーション付きUIボタンが作れます。

## 💡 Tips

### スライスの表示切り替え

**View > Show Slices** でキャンバス上のスライス境界の表示/非表示を切り替えられます。作画中は非表示にしておくと邪魔になりません。

### 角サイズの目安

| UI素材サイズ | 推奨角サイズ |
| ------------ | ------------ |
| 16×16〜24×24 | 2〜4px       |
| 32×32〜48×48 | 4〜6px       |
| 64×64以上    | 6〜8px       |

角が大きすぎると引き伸ばし時に中央が細くなります。小さすぎると角のディテールが表現できません。

### UI アトラスの作り方

ゲームのUI素材をすべて1つのスプライトにまとめて、それぞれにスライスを設定する運用がおすすめです。ファイル管理が楽になり、テクスチャの切り替えも減るのでパフォーマンスにも良い影響があります。
